ui-vmm: how it works
Working with servers is built using the so-called providers.
The provider implements work with servers and sends the ui-vmm result in a prepared format.
ui-vmm
provider
* Since ui-vm works with providers via gRCP, you can create your provider by taking the protobuf schema from the ui-vmm repository.
providers and their work with servers
First of all, the provider launches the gRPC server to which ui-vmm connects.
Next, the provider requests data from the server and sends the formatted result via gRPC.
provider
* ui-vmm can work with both lxd provider and kvm or ldap.
queues for asynchronous operation
For asynchronous work ui-vmm uses a solution from Skytools - PgQ.

Easy to Use

To work, you just need to install ui-vmm and connect it to providers. For example, it could be the lxd or ldap provider.

Expandability

UI-vmm functionality is being expanded by providers. You can take ready-made providers or write your own by taking the protobuf scheme from the repository.

Used protocols and extensions

Using gRCP and PgQ makes it possible to work asynchronously with external providers. It's really cool ;)
